# npm Deep Dive

## 챕터별 학습 목표

### 1장: npm과 유의적 버전

- [ ] npm의 정의에 대해 설명 가능하다.
- [ ] npm의 역사와 배경을 설명할 수 있다.
- [ ] npm의 주요 기능을 나열하고 설명할 수 있다.
- [ ] npm 레지스트리의 구조를 설명할 수 있다.
- [ ] 유의적 버전에 대해 알고 프로젝트에 버전 관리를 한다면 유의적 버전을 의도에 맞게 적용할 수 있다.
- [ ] 유의적 버전의 등장 배경과 정의를 설명할 수 있다.
- [ ] 유의적 버전의 구체적인 명세를 설명할 수 있다.
- [ ] 유의적 버전의 문법(^, ~, *, >=, <= 등)을 설명할 수 있다.
- [ ] Node.js와 npm에서 사용하는 유의적 버전을 설명할 수 있다.
- [ ] left-pad 사건의 배경과 영향력을 설명할 수 있다.
- [ ] is-promise 사태의 원인과 해결 과정을 설명할 수 있다.
- [ ] colors.js와 faker.js 사건을 통해 배운 교훈을 설명할 수 있다.
- [ ] event-stream 사건의 보안 이슈를 설명할 수 있다.
- [ ] 유의적 버전과 npm을 사용할 때 주의할 점을 나열하고 설명할 수 있다.
- [ ] package-lock.json의 역할을 설명할 수 있다.

### 2장: npm 기본기 다지기 - package.json과 npm 명령어 완전 정복

- [ ] 책의 2.1 package.json 톺아보기를 보고, package.json에 대해 설명 가능 (블로그 포스팅)
- [ ] package.json의 구조와 각 필드의 의미를 설명할 수 있다.
- [ ] name, version, description, main, scripts, keywords 등의 필드를 설명할 수 있다.
- [ ] 2.2 dependencies란 무엇일까?를 보고, 각각의 의존성에 대해 이해하고 설명할 수 있다.
- [ ] dependencies, devDependencies, peerDependencies, optionalDependencies의 차이를 설명할 수 있다.
- [ ] 각 의존성 타입을 언제 사용해야 하는지 판단할 수 있다.
- [ ] npm 명령어의 종류와 사용법을 설명할 수 있다.
- [ ] npm install, npm uninstall, npm update, npm audit 등의 명령어를 설명할 수 있다.
- [ ] 스크립트와 라이프사이클을 설명할 수 있다.
- [ ] npm scripts를 사용한 빌드 자동화를 설명할 수 있다.
- [ ] 의존성 관리와 버전 범위를 설명할 수 있다.
- [ ] 패키지 배포와 버전 관리 방법을 설명할 수 있다.
- [ ] npm publish 과정을 설명할 수 있다.
- [ ] .npmignore 파일의 역할을 설명할 수 있다.

### 3장: node_modules 이해하기

- [ ] 의존성 해결 과정을 설명할 수 있다.
- [ ] 의존성 트리의 개념을 설명할 수 있다.
- [ ] npm, yarn, pnpm의 차이점을 알고 각각의 특징을 설명할 수 있다.
- [ ] 각 패키지 매니저의 의존성 설치 전략을 설명할 수 있다.
- [ ] hoisting의 개념을 설명할 수 있다.

- [ ] pnpm의 심볼릭 링크 방식의 장점을 설명할 수 있다.
