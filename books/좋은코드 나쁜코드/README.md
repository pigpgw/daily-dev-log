# 좋은코드 나쁜코드

## 챕터별 학습 목표

### 1장: 좋은 코드와 나쁜 코드의 정의

- [ ] 좋은 코드의 기준을 설명할 수 있다.
- [ ] 나쁜 코드의 특징을 나열하고 설명할 수 있다.
- [ ] 코드 품질이 프로젝트에 미치는 영향을 설명할 수 있다.
- [ ] 코드 리뷰 시 체크할 기본 원칙을 설명할 수 있다.

### 2장: 가독성

- [ ] 가독성 있는 코드를 작성하는 원칙을 설명할 수 있다.
- [ ] 네이밍 규칙을 설명할 수 있다.
- [ ] 변수명, 함수명, 컴포넌트명 네이밍 컨벤션을 설명할 수 있다.
- [ ] 코드 구조화 방법을 설명할 수 있다.
- [ ] 주석 작성 원칙을 설명할 수 있다.
- [ ] 매직 넘버/문자열을 상수로 추출하는 이유를 설명할 수 있다.

### 3장: 재사용성과 모듈화

- [ ] 재사용 가능한 코드를 작성하는 방법을 설명할 수 있다.
- [ ] 모듈화의 원칙을 설명할 수 있다.
- [ ] DRY(Don't Repeat Yourself) 원칙을 설명하고 적용할 수 있다.
- [ ] 공통 컴포넌트 설계 원칙을 설명할 수 있다.
- [ ] 유틸리티 함수 설계 방법을 설명할 수 있다.
- [ ] 추상화 레벨을 적절히 유지하는 방법을 설명할 수 있다.

### 4장: 테스트 가능성

- [ ] 테스트하기 쉬운 코드 구조를 설명할 수 있다.
- [ ] 의존성 주입의 중요성을 설명할 수 있다.
- [ ] 단위 테스트 작성 방법을 설명할 수 있다.
- [ ] 순수 함수의 중요성을 설명할 수 있다.
- [ ] Mock과 Stub의 차이를 설명할 수 있다.
- [ ] 테스트 커버리지의 의미를 설명할 수 있다.

### 5장: 유지보수성

- [ ] 유지보수하기 쉬운 코드를 작성하는 방법을 설명할 수 있다.
- [ ] 리팩토링 원칙을 설명할 수 있다.
- [ ] 코드 복잡도를 관리하는 방법을 설명할 수 있다.
- [ ] 순환 복잡도(Cyclomatic Complexity)를 설명할 수 있다.
- [ ] 함수/컴포넌트의 크기 기준을 설명할 수 있다.
- [ ] 변경에 유연한 코드를 작성하는 방법을 설명할 수 있다.

### 6장: 성능 최적화

- [ ] 성능에 영향을 주는 코드 패턴을 설명할 수 있다.
- [ ] 성능 최적화 기법을 설명할 수 있다.
- [ ] 성능과 가독성의 트레이드오프를 설명할 수 있다.
- [ ] 불필요한 연산을 피하는 방법을 설명할 수 있다.
- [ ] 메모이제이션을 언제 사용해야 하는지 판단할 수 있다.
- [ ] 성능 측정 도구를 사용하는 방법을 설명할 수 있다.

### 7장: 보안

- [ ] 보안 취약점이 있는 코드 패턴을 설명할 수 있다.
- [ ] XSS(Cross-Site Scripting) 공격을 방지하는 방법을 설명할 수 있다.
- [ ] 안전한 코딩 기법을 설명할 수 있다.
- [ ] 입력 검증과 출력 인코딩의 중요성을 설명할 수 있다.
- [ ] 민감한 정보를 안전하게 처리하는 방법을 설명할 수 있다.
- [ ] 의존성 보안 취약점을 관리하는 방법을 설명할 수 있다.

### 8장: 실제 사례 분석

- [ ] 좋은 코드와 나쁜 코드의 실제 예시를 비교 분석할 수 있다.
- [ ] 코드 리뷰 시 체크할 포인트를 설명할 수 있다.
- [ ] 리팩토링 전후를 비교하고 개선점을 설명할 수 있다.
- [ ] 팀 내 코드 품질을 유지하는 방법을 설명할 수 있다.
