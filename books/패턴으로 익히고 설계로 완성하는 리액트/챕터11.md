## 핵심 내용

### 클래스 기반의 모델 장점

-   캡슐화 : 클래스는 관련된 속성과 메서드를 한곳에 위치하게 하여 깔끔한 구조화가 가능하다. 그리고 데이터 접근을 제한할 수 있어 제어하기 쉽고, 데이터 무결성도 유지할 수 있음
-   메서드 : 메뉴 아이템과 관련된 복잡한 작업이 있을때, 클래스가 제공하는 구조를 활용해 데이터 조작이든 별도의 비지니스 로직이든 상관없이 메서드를 정의할 수 있음
-   상속과 다형성 : 메뉴 아이템에서 상속이나 다형성 개념이 필요할 때 클래스 구조는 필수적입, 각각 다른 메뉴 아이템 타입을 공통으 ㅣ기본 클래스에서 상속받아 필요한 행동을 재정의할 수 있음
-   일관된 인터페이스 : 클래스는 특히 여러 애플리케이션 영역에서 메뉴 아이템을 다룰 때 데이터의 일관된 인터페이스를 보장
-   읽기 전용 속성 : 클래스는 읽기 전용 속성을 정의할 수 있으므로 데이터 수정을 제한할 수 있음, 데이터 무결성을 유지하고 불변 데이터 구조에서 작업하기 위한 필수적인 요소
