## 알고리즘

1. **문제 목표 분석**
    - 무엇을 구하는가? (최댓값 / 최솟값 / 경우의 수 / 존재 여부 / 경로)
    - 모든 원소를 사용하는가?
    - 순서를 바꿀 수 있는가?
    - 부분 선택이 가능한가?
    - 최적화 문제인가, 단순 구현인가?

2. **입력 제한 확인 (복잡도 상한 결정)**
    - N, M을 10의 n승 감각으로 변환
    - 1초 ≈ 10⁸ 연산 기준 적용
    - 가능한 시간복잡도 범위 추리
        - N ≤ 10 → O(N!) 가능
        - N ≤ 20 → O(2^N) 가능
        - N ≤ 1000 → O(N²) 가능
        - N ≤ 10⁵ → O(N log N) 필요
        - N ≥ 10⁶ → O(N) 이하 필요

3. **완전탐색 / 백트래킹 가능 여부 판단**
    - 모든 경우를 다 해봐도 되는가?
    - 순열 / 조합 / 부분집합 문제인가?
    - 가지치기가 필요한 구조인가?

4. **그리디 가능 여부 판단**
    - 정렬 후 해결될 가능성이 있는가?
    - “지금 최선 선택”이 항상 전체 최선인가?
    - 반례가 바로 떠오르는가?
    - 증명 가능성이 있는가?

5. **DP 구조 존재 여부 판단**
    - 작은 문제로 나눌 수 있는가?
    - 상태 정의가 가능한가? (dp[i], dp[i][j] 등)
    - 이전 결과를 재사용하는가?
    - 중복 계산이 발생하는가?

6. **그래프 / 자료구조 유형 확인**
    - 연결 관계가 핵심인가? → BFS / DFS
    - 최단 거리인가? → BFS / 다익스트라
    - 위상 관계인가? → 위상정렬
    - 구간 문제인가? → 세그트리 / 누적합
    - 스택/큐/우선순위 큐가 자연스러운가?
